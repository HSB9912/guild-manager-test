# Requirements: 뚠카롱 길드 관리 시스템

**Defined:** 2026-03-24
**Core Value:** 길드 운영에 필요한 모든 데이터를 한곳에서 빠르고 정확하게 관리할 수 있어야 한다.

## v1 Requirements

### Foundation (프로젝트 기반)

- [ ] **FOUND-01**: React + Vite + TypeScript 프로젝트가 초기화되고 빌드/개발 서버가 동작한다
- [ ] **FOUND-02**: Tailwind CSS 4가 PostCSS로 설정되고 기존 클래스명이 동일하게 동작한다
- [ ] **FOUND-03**: React Router로 페이지 간 네비게이션이 동작한다 (홈, 길드원, 분석, 승강제, 현황, 수로보상, 게시판, 버디)
- [ ] **FOUND-04**: 사이드바 + 헤더 + 모바일 탭바 레이아웃이 반응형으로 동작한다
- [ ] **FOUND-05**: 다크모드 토글이 동작하고 모든 컴포넌트에 적용된다
- [ ] **FOUND-06**: Supabase 클라이언트가 단일 인스턴스로 초기화된다
- [ ] **FOUND-07**: 폰트 크기 설정 (소/중/대)이 동작한다

### Authentication (인증)

- [ ] **AUTH-01**: 관리자가 Google OAuth로 로그인할 수 있다
- [ ] **AUTH-02**: 허용된 이메일만 접근 가능하고, 미허용 계정은 거부 화면을 본다
- [ ] **AUTH-03**: 세션이 브라우저 새로고침 후에도 유지된다
- [ ] **AUTH-04**: 로그아웃 후 인증 화면으로 돌아간다
- [ ] **AUTH-05**: Public View 모드에서 비로그인 사용자가 읽기 전용으로 접근할 수 있다

### Members (길드원 관리)

- [ ] **MEMB-01**: 관리자가 길드원을 추가할 수 있다 (닉네임, 직업, 레벨, 랭크, 가입일)
- [ ] **MEMB-02**: 관리자가 길드원 정보를 수정할 수 있다
- [ ] **MEMB-03**: 관리자가 길드원을 삭제할 수 있다
- [ ] **MEMB-04**: 길드원 목록에서 검색/필터/정렬이 동작한다
- [ ] **MEMB-05**: 길드원 일괄 수정(배치 편집)이 동작한다

### Scores (수로 점수)

- [ ] **SCORE-01**: 관리자가 주차별 수로 점수를 기록할 수 있다
- [ ] **SCORE-02**: 주차별 전체 점수 목록을 테이블로 조회할 수 있다
- [ ] **SCORE-03**: 미참여(0점) 길드원이 시각적으로 강조 표시된다
- [ ] **SCORE-04**: 점수 테이블 컬럼별 정렬이 동작한다
- [ ] **SCORE-05**: 주차 선택/전환이 동작한다

### Analysis (수로 분석)

- [ ] **ANAL-01**: 개인별 수로 점수 추이 차트가 표시된다
- [ ] **ANAL-02**: 길드 전체 참여율 통계가 표시된다
- [ ] **ANAL-03**: 주간/월간 점수 랭킹이 표시된다
- [ ] **ANAL-04**: MVP/우수 길드원 하이라이트가 동작한다
- [ ] **ANAL-05**: 분석 요약 테이블이 동작한다

### Promotion (승강제)

- [ ] **PROMO-01**: 승급/강등 기준 점수 임계값을 설정할 수 있다
- [ ] **PROMO-02**: 기준에 따른 승급/강등 대상자가 자동 계산된다
- [ ] **PROMO-03**: 승강 이력이 기록되고 조회 가능하다
- [ ] **PROMO-04**: 승강 대상자 편집/확정이 동작한다

### Guild Stats (현황)

- [ ] **STAT-01**: 길드원 레벨 분포 차트가 표시된다
- [ ] **STAT-02**: 직업 분포 차트가 표시된다
- [ ] **STAT-03**: 길드 전체 통계 요약 (총원, 평균 레벨 등)이 표시된다
- [ ] **STAT-04**: 통계 대시보드가 기존 대비 강화된 형태로 표시된다

### Rewards (수로 보상)

- [ ] **REWARD-01**: 수로 점수 기반 보상이 자동 계산된다
- [ ] **REWARD-02**: 보상 내역을 조회할 수 있다
- [ ] **REWARD-03**: 보상 계산 공식/설정을 변경할 수 있다

### Board (게시판)

- [ ] **BOARD-01**: 관리자가 게시글을 작성할 수 있다 (제목, 내용, 이미지)
- [ ] **BOARD-02**: 게시글 목록을 조회할 수 있다
- [ ] **BOARD-03**: 게시글 상세 보기가 동작한다
- [ ] **BOARD-04**: 게시글 수정/삭제가 동작한다
- [ ] **BOARD-05**: 이미지 업로드가 Cloudflare R2를 통해 동작한다

### Buddy (뚠뚠 버디)

- [ ] **BUDDY-01**: 버디 매칭을 실행할 수 있다
- [ ] **BUDDY-02**: 매칭 결과를 조회할 수 있다
- [ ] **BUDDY-03**: 이전 매칭 이력이 보존되어 중복 방지된다

### Calendar (캘린더)

- [ ] **CAL-01**: 월별 캘린더가 표시되고 월 이동이 동작한다
- [ ] **CAL-02**: 일정(이벤트)을 추가/수정/삭제할 수 있다
- [ ] **CAL-03**: 목요일(수로 마감일)이 시각적으로 강조된다
- [ ] **CAL-04**: 일정 상세 보기가 동작한다

### OCR (스크린샷 인식)

- [ ] **OCR-01**: 메이플스토리 스크린샷을 업로드하면 점수가 자동 인식된다
- [ ] **OCR-02**: 다양한 해상도(1366x768 ~ 3840x2160)를 지원한다
- [ ] **OCR-03**: 인식 결과를 수정 후 점수에 반영할 수 있다
- [ ] **OCR-04**: OpenCV.js WASM이 React 환경에서 정상 로딩된다

## v2 Requirements

### Enhanced Dashboard

- **DASH-01**: CSV/이미지 내보내기 (디스코드 공유용)
- **DASH-02**: 커스텀 대시보드 위젯 배치

### Notifications

- **NOTF-01**: Discord 웹훅으로 공지 알림
- **NOTF-02**: 수로 마감 리마인더

## Out of Scope

| Feature | Reason |
|---------|--------|
| 실시간 채팅 | 카톡/디스코드와 중복, 높은 복잡도 대비 낮은 채택률 |
| 보스 레이드 스케줄러 | 별도 제품 수준의 범위, 캘린더로 최소 충족 |
| PWA / 오프라인 모드 | 관리자 작업은 항상 온라인, 서비스워커 복잡도 대비 이점 없음 |
| Nexon Open API 연동 | API 제한/유지보수 부담, OCR이 이미 해결 |
| 멀티 길드 / SaaS 모드 | 뚠카롱 전용 도구, 범용화는 범위 밖 |
| 공개 멤버 포털 | 관리자 전용 도구, 접근제어 복잡도 증가 |
